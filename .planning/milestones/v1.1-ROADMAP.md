# Milestone v1.1: Validation, Benchmarking, and Distribution

**Status:** ✅ SHIPPED 2026-03-17

## Overview

v1.1 proved the shipped OptimusCtx runtime end to end, measured workflow and token savings with rerunnable benchmark evidence, repaired the counted-input benchmark boundary, and turned distribution into a credible release story across GitHub Releases, Homebrew, Scoop, npm, and `npx`.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Functional and benchmark evidence stays fixture-backed, rerunnable, and stored under repo-local artifacts rather than summary-only prose.
- Benchmark claims count only declared agent-facing inputs and require comparable final artifacts before a lane is considered successful.
- GitHub Releases remains the canonical binary source, while Homebrew, Scoop, npm, and `npx` stay truthful wrapper or packaging channels around the same shipped runtime.

**Issues Resolved:**

- Added end-to-end CLI and MCP validation with persisted evidence for healthy, stale, degraded, and recovery scenarios.
- Converted benchmark methodology into reproducible machine-readable and human-readable evidence, then repaired the counted-input boundary.
- Added a credible operator distribution path through release archives, first package-manager channels, install verification, and npm publication.

**Issues Deferred:**

- Secondary tokenizer metrics remain future benchmark depth work.
- Watch-assisted edit-loop benchmarking remains deferred until the non-watch baseline stays stable.
- Native Linux packages, signed artifacts, and SBOM metadata remain future distribution expansion work.

**Technical Debt Incurred:**

- `gsd-tools` milestone auto-detection still resolves `v1.0` unless `v1.1` is passed explicitly during closeout workflows.
