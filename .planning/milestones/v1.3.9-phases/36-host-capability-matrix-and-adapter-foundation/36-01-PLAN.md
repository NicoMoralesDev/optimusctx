# Plan 36-01: Host Capability Metadata

## Objective

Add explicit capability metadata for first-class hosts and remove positional/ad hoc host selection from onboarding and diagnostics code paths.

## Tasks

- Extend `repository.SupportedClient` with capability metadata that explains config format, scope model, guidance support, usage evidence, and mixed-environment handling.
- Replace positional client indexing in install and doctor flows with explicit lookup by client ID.
- Add tests that lock the host catalog and capability truth for the currently shipped first-class hosts.
